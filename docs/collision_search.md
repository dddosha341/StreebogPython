# `src/collision_search.py` — birthday attack для `MSB48`

Проект намеренно использует **усечённую** хэш-функцию:

`h48(x) = MSB48(STREEBOG-512(x))`  
где `MSB48` берётся как первые 6 байтов дайджеста в реализации проекта (`digest[:6]`).

Цель модуля — найти две разные входные строки `msg1 != msg2`, такие что:

`h48(msg1) == h48(msg2)`

## Основные сущности

- `h48(data) -> bytes`
  - вычисляет `Streebog-512` через `hash_bytes(..., impl="fast")` и возвращает `digest[:6]`.

- `find_collision(seed=None, msg_size=16, max_attempts=50_000_000, log_interval=1_000_000, out_dir=None)`
  - реализует birthday attack:
    - генерирует случайные сообщения
    - хранит в словаре `seen: h48 -> message`
    - при совпадении префикса проверяет, что сообщения различны
  - при `out_dir` сохраняет результат в `collision.json`.

## Выходные данные

Функция возвращает:

- `(msg1, msg2, hash_prefix, attempts, elapsed_seconds)`

Если задан `out_dir`, создаётся директория и файл:

- `collision.json` с hex-представлениями сообщений и дайджестов, числом попыток и оценкой ожидаемого числа попыток.

## Примечание по стоимости

По парадоксу дней рождения для 48 бит ожидаемое число попыток порядка `~1.25 * 2^24` (десятки миллионов), поэтому поиск может быть длительным.
