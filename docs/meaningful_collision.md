# `src/meaningful_collision.py` — осмысленная коллизия (2 BMP)

Этот модуль строит «осмысленную» коллизию для:

`h(x) = MSB48(STREEBOG-512(x))`

Вместо двух случайных сообщений он создаёт два **разных валидных BMP**-файла, которые визуально различаются, но имеют совпадающее значение `h48`.

## Идея

1. Генерируются два 24-битных несжатых BMP изображения 64x64:
   - `image1_X.bmp` — паттерн «X»
   - `image2_O.bmp` — паттерн «O»
2. К пиксельным данным дописываются случайные байты (суффиксы).
3. При просмотре BMP обычно учитывается объявленная «правильная» область пикселей, поэтому суффикс не виден.
4. При хэшировании учитывается весь файл, включая суффикс.

## Основные функции

- `_create_bmp(width, height, pixels_func) -> bytes`
  - формирует байты BMP (headers + pixel data), строки идут снизу вверх как в BMP-спецификации

- `find_meaningful_collision(out_dir="data/output", suffix_size=32, max_attempts=50_000_000, log_interval=500_000)`
  - реализует поиск коллизии между двумя множествами суффиксов по birthday attack
  - возвращает `(image_a_bytes, image_b_bytes, h48_value)`

## Оптимизация

Для ускорения используется предвычисление:

- `_precompute_hash_state(image_data)` обрабатывает все полные 64-байтовые блоки изображения и возвращает потоковое состояние hasher + остаток неполного хвоста.
- `_hash_with_suffix(base_hasher, remaining, suffix)` клонирует состояние (`copy()`), дообрабатывает `remaining + suffix` и финализирует.

За счёт этого на попытку не пересчитываются полные изображения.

## Сохранение результата

При нахождении совпадения создаётся JSON-файл `meaningful_collision.json` в `out_dir`, а также записываются:

- `image1_X.bmp`
- `image2_O.bmp`

В метаданных сохраняются:
`h48`, полные хэши, число попыток, время, и описание.
